# Security policy

Report vulnerabilities privately through GitHub's **Report a vulnerability**
flow rather than a public issue. Include the affected tag, printer/board model,
and a minimal reproduction. The latest `0.2.x` beta accepts security reports;
maintenance and community support are best-effort during the beta.

This component controls physical output. Treat malformed-device handling,
unbounded waits, duplicate-label behavior, and quarantine bypasses as security
or safety issues even when they do not expose network data.
